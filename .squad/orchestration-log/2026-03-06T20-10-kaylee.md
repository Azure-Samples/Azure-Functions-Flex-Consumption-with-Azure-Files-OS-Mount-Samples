### Objective
Rewrite all shared Bicep infrastructure modules to correctly deploy Azure Functions Flex Consumption apps, using the official reference from [Azure-Samples/azure-functions-flex-consumption-samples](https://github.com/Azure-Samples/azure-functions-flex-consumption-samples).

### Modules Rewritten

#### 1. **function-app.bicep** 
- **Status:** ✅ Complete
- **Key changes:**
  - Added `functionAppConfig` property with:
    - `deployment.storage` for blob container + managed identity auth (mandatory for Flex Consumption)
    - `scaleAndConcurrency` with `maximumInstanceCount` and `instanceMemoryMB`
    - `runtime` with language name and version
  - Enabled system-assigned managed identity (`type: 'SystemAssigned'`)
  - Replaced raw `AzureWebJobsStorage` connection string with:
    - `AzureWebJobsStorage__credential: 'managedidentity'`
    - Separate `AzureWebJobsStorage__blobServiceUri`, `AzureWebJobsStorage__queueServiceUri`, `AzureWebJobsStorage__tableServiceUri`
  - Removed legacy properties: `linuxFxVersion`, `FUNCTIONS_WORKER_RUNTIME`, `FUNCTIONS_EXTENSION_VERSION`, `AzureWebJobsFeatureFlags`, `APPINSIGHTS_INSTRUMENTATIONKEY`
  - Added `APPLICATIONINSIGHTS_AUTHENTICATION_STRING: 'Authorization=AAD'`
  - Outputs now include `principalId` for RBAC role assignments
  - API version bumped to `2024-04-01`

#### 2. **storage-account.bicep**

### Technical Rationale
1. `functionAppConfig` is the **only way** to create a true Flex Consumption app—without it, the app deploys as a standard Consumption plan
2. Managed identity is the recommended authentication pattern for Flex Consumption; shared key connection strings are deprecated
3. Deployment via blob container (not zip) is the standard Flex Consumption pattern
4. `linuxFxVersion` is a legacy property; Flex Consumption uses `functionAppConfig.runtime` instead
5. All changes align with the official Azure reference implementation
